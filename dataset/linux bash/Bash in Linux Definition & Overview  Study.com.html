Bash in Linux: Definition & Overview

In this lesson, we will explore the bash shell in Linux. We will discover the role of file descriptors and standard input and output. Lastly, we will learn the meaning of redirection using pipes, a powerful mechanism to run complex commands.

Knowing the Language

If you want to speak to someone who doesn't know your language, you will need an interpreter to translate the conversation between you and the other person. Similarly, if you want to run a command on a computer, the computer also needs an interpreter to understand the command and execute it.

Shells

On Linux, the interpreter that can understand the user commands and settings and pass them on to the computer for further processing is called a shell. Here are a few shells on Linux:

  • sh - The Bourne shell is the first shell to be introduced on UNIX. Bash improves upon sh. All the sh commands can also be run on bash, but not all bash commands can be used on the sh shell. This makes bash a superset of sh and other options/modules.
  • csh - The C shell with a syntax similar to the C programming language.
  • tcsh - The TENEX C shell is the C shell with additional features.
  • ksh - The Korn shell, a command and programming language.

On most Linux systems, bash, called the Bourne Again SHell, is set as the default shell. The bash shell is also responsible for letting the users configure a set of user preferences or settings, also known as a user environment.

To know if bash exists on a Linux system, run the following command to display the contents of the /etc/shells file:

cat /etc/shells

The output will look something like this. It is a list of all the shells available on the system:

/bin/bash

/bin/sh

/bin/csh

/bin/tcsh

/bin/ksh

Script

Apart from the commands you enter, bash can run a set of commands from a file, called a shell script, which we will refer to as a script going forward. Thus, bash is also a programming language in its own right. It contains its own syntax and programming constructs such as break, continue, exit, exec, return, and so on.

File Descriptors and Standard I/O

In Linux, everything is a file, be it a regular file with text in it or a device file. As a shell, bash regularly deals with three common files for standard input and output (I/O):

  • stdin - Keyboard, the standard input used for reading user inputs, including commands.
  • stdout - Screen, the standard output used to display/write the output of a command.
  • stderr - Screen, the standard output used to display the error messages of a command.

By default, these files are always open. Whenever a file is open in Linux, it is assigned a number called a file descriptor to keep track of it. If you are familiar with the C programming language, a file descriptor is similar to a file handle or a pointer.

Thus, stdin, stdout, and stderr are assigned the file descriptors 0, 1, and 2 respectively. In the bash shell commands and scripts, they standard I/O are referred to as fd 0, fd 1, and fd 2, respectively. When other files are open, they get assigned the file descriptors between 3 and 9.

Redirections and Pipes

Imagine a simple command:

cat file.txt

In this example, cat is the command followed by file.txt as the argument for the command. This is an example of a simple command.

In reality, you might have to run a sequence of complex commands that includes a set of simple commands. Typically, if we had to bring all of them together to run in a particular sequence, you would have to do put the commands in a programming construct such as a loop or conditions.

In Linux, you can use a method called as redirection by putting the commands in a pipeline. You can use pipes (|) to gather the output of one command and redirect it as the input of another command.

Example 1

cat file.txt | more

The command cat file.txt displays the contents of the file.txt on the standard output, the screen. When its output is redirected to the more command, the contents of the file.txt file are displayed one screen at a time. That's the power of redirection using pipes!

Imagine you have a gunzipped file of a tar file. To extract the actual contents of the file, you would have to run two commands to first extract the tar file and then extract the tar file, as follows:

  • gunzip file.tar.gz
  • tar file.tar xvf -

With the simplicity and the elegance of redirection, the same result can be obtained using the following command:

gunzip file.tar.gz | tar xvf -

Therefore, redirection helps you to gather the output from a script, a program, a command, or a file and redirect it as input to another script, a program, a command, or a file.

Example 2

We learned that each command is associated with three standard I/O files - a stdin (fd 0), stdout (fd 1), and a stderr (fd 2).

If you want to send the stderr output to stdout, run the following command:

ls 2>&1

The &1 notation refers to the address of fd 1, stdout.

Sometimes, you might want to close a file descriptor for a specific reason or after a job is done.

For example, to close stdout, run the following command:

1>&-, >&-

Lesson Summary

In this lesson, we discovered the Linux shells and explored the bash shell.

We learned that a bash command is attached to three standard input and output (I/O) files:

  • stdin
  • stdout
  • stderr

Further, they are assigned the file descriptors of 0, 1, and 2 respectively. Any other open file is assigned a file descriptor between 3 and 9.

Through examples, we learned redirection through pipes, which is a method to redirect the input from one command/script as the input of another.
